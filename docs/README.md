## 📖 Cómo Usar Esta Documentación

### Si eres nuevo en el proyecto:
1. Comienza con [DEVELOPMENT.md](DEVELOPMENT.md) para configurar tu entorno
2. Lee [ARCHITECTURE.md](ARCHITECTURE.md) para entender la estructura
3. Consulta [API.md](API.md) para conocer los endpoints disponibles

### Si vas a contribuir:
1. Lee [../CONTRIBUTING.md](../CONTRIBUTING.md) para conocer las reglas
2. Revisa [../TESTING.md](../TESTING.md) para escribir tests
3. Sigue las convenciones en [DEVELOPMENT.md](DEVELOPMENT.md)

### Si vas a desplegar:
1. Consulta [DEPLOYMENT.md](DEPLOYMENT.md) para configuración de producción
2. Revisa las variables de entorno necesarias
3. Configura monitoreo y backups

## 🔄 Mantenimiento de la Documentación

Esta documentación debe mantenerse actualizada con cada cambio significativo:

- **Nuevos endpoints**: Actualizar [API.md](API.md)
- **Cambios arquitectónicos**: Actualizar [ARCHITECTURE.md](ARCHITECTURE.md)
- **Nuevas herramientas**: Actualizar [DEVELOPMENT.md](DEVELOPMENT.md)
- **Cambios en despliegue**: Actualizar [DEPLOYMENT.md](DEPLOYMENT.md)
